O conflito foi gerado a partir do momento que eu criei duas branches(projetoA e projetoB) a partir do mesmo commit. Porém, durante a execução do código da projetoA, eu esqueci de dar o comando "git push" para jogar o código do meu repositório local para o online. Então quando fui fazer o merge das duas branches teoricamente conflitantes, não deu erro e ficou como versão certa o projetoB, que teoricamente não seria a correta. Quando percebi o erro, voltei para a branch projetoA e fiz o código novamente, dessa vez dei o "git push" e, no momento de fazer o merge, deu que a branches eram conflitantes. Para resolver o problema, alterei manualmente o projetoB para que ele fosse apagado e manter somente o projetoA. O projetoB fazia a introdução à sede do Santa Cruz, mas todos sabemos que o maior clube de Recife é o Náutico, por isso foi mantida a branch projetoA.
