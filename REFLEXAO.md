Inicialmente, eu tive muita dificuldade, pois nunca havia usado os comandos do Git e tive dificuldade para fazer os commits. Essa minha dificuldade é perceptível pelo horário dos meus commits, eu comecei num dia, mas me estressei muito porque não conseguia dar commit numa mudança de texto de um arquivo. Então deixei para fazer no dia seguinte e consegui bem mais rápido, estava esquecendo registrar as mudanças para o commit ser válido. A minha segunda dificuldade foi no exercício 3, na hora de criar o conflito, esqueci de dar git push para atualizar o código no repositório online, então na hora que era pra dar o conflito, apareceu que era possível dar merge automaticamente. Então tive que voltar na branch inicial para alterar o código e dar o pull request novamente. Na hora de resolver o conflito foi bem simples, eu já sabia que uma das branches estava errada, então foi só apagar e dar merge.
Eu gostei muito de usar os comandos no terminal, antes eu usava só a interface do VS Code, mas dessa vez me forcei a aprender os códigos. Também aprendi esses commits semânticos que ajudam na hora de entender qual foi a mudança feita em cada parte do projeto, além, obviamente, das nomenclaturas das branches para evitar que duas pessoas façam a mesma coisa ao mesmo tempo.
Para o futuro, preciso botar em prática pra não esquecer, e entender um pouco mais sobre as mudanças dentro dos arquivos porque ainda está um pouco confuso sobre quando eu dou "git add" e quando eu dou commit direto.
Apenas um breve comentário sobre a atividade, no início eu tava odiando fazer, porque achei que era totalmente inútil e só tava me dando estresse. Mas quando eu consegui resolver os problemas, percebi que foi muito útil. Só acho que poderia diminuir um pouco a quantidade de palavras da reflexão, antes de eu começar a escrever essa frase, meu texto tava com 298 palavras e eu já tinha falado sobre tudo porque é uma atividade pequena pra se falar tanto.
